目录结构

Tower/game.py

展开 game.py
展开game.py中的road_map
其中踩了大坑的难点😭

game.py控制着所有逻辑,其实没有什么难的地方.
至于为什么要用road_map…这个有意思了…😂
(road_map是另外用函数写的把背景整个地图做成二维数组)
本来我很简单顺水推舟地使用enemy的行走路径point[ ]来判断不可以把tower建在怪物行走路径上面.



然后…这样就涉及到两点形成了一条直线,然后直线外一点(鼠标位置),到直线的最短距离d不少于road的1/2宽.

我们有任意三个点:

鼠标(x, y) 路径一个点(x1, y1), 路径下一个点(x2, y2)

想知道公式推导的话去看李永乐老师的推导…
python中利用math计算

a = y2 - y1
b = x1 - x2
c = x2 * y1 - x1 * y2
d = (math.fabs(a * x + b * y +c )) / (math.pow (a * a + b * b , 0.5))

方法是有了,但是😭…运行起来十分吃力(卡顿卡顿卡顿)…pygame还是适合轻量级游戏,一边大量绘画,一边大量计算,真的不合适…,让我产生了一丝…😭python的确不合适做游戏…

然而天生不是这料.没关系.那就换个方法.(许多天以后…)
利用二维数列形成了虚拟地图,把地图分成了每个像素为25的小box.
鼠标获取的pos/25就是二维地图中的box,用0和1来判断是与否.快速,节省资源,效果让我很满意.


然后就是,代码量上千行以后,越写到后面,我回头一看想改点东西…咦? 😨我这里定义的是什么意思…这里又是什么意思…

注释很重要,写给自己看的😂
